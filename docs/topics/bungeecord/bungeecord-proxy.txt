[b]Description:[/b]
EzEconomy - BungeeCord Proxy is the proxy connector for EzEconomy that synchronizes balances and account actions across a BungeeCord network. It provides a fast, reliable bridge between your proxy and backend storage (MySQL, SQLite, MongoDB, Redis) and integrates with Vault and PlaceholderAPI so servers and plugins can access a single unified economy.

[b]Key Features[/b]
[list]
[*]Cross-server shared balances for BungeeCord networks
[*]Supports MySQL, SQLite, MongoDB and Redis backends
[*]Vault-compatible for broader plugin support
[*]PlaceholderAPI expansion for easy placeholders
[*]Asynchronous storage operations and caching for performance
[*]Simple configuration and minimal permissions surface
[*]Works with Minecraft versions 1.7.10 — 1.21.x (server compatibility depends on backend modules)
[/list]

[b]Requirements[/b]
[list]
[*]BungeeCord or a fork-compatible proxy
[*]A backend storage provider (MySQL/Redis/etc.) configured in EzEconomy core
[*]Java 8+ on proxy host
[/list]

[b]Installation[/b]
[list]
[*]1) Place the EzEconomy proxy jar into your proxy's `plugins` folder.
[*]2) Restart or reload the proxy.
[*]3) Configure the connection in the core EzEconomy plugin (on a backend server) and ensure storage provider settings are reachable from the proxy.
[*]4) Verify the proxy shows as connected in the EzEconomy core logs.
[/list]

[b]Locking Strategy[/b]
[list]
[*][b]Default:[/b] LOCAL (in-JVM locking). No extra setup required.
[*][b]Options:[/b] LOCAL, REDIS, BUNGEECORD (set `locking-strategy` in `config.yml`).
[*][b]Use REDIS when:[/b] you run multiple servers sharing the same economy state and need cross-instance synchronization.
[*][b]Safety notes:[/b] If REDIS is selected but cannot initialize, `fallback-to-local` (in `redis.yml`) controls fallback behavior. Transfers acquire locks in a deterministic UUID order to avoid deadlocks. Changing the strategy requires a plugin restart.
[*][b]Enable Redis lock guide:[/b]
	[list]
	[*]1) Install `ezeconomy-redis.jar` in `plugins/EzEconomy/libs/` on servers that need Redis.
	[*]2) Set `locking-strategy: REDIS` in `config.yml`.
	[*]3) Configure `redis.yml` (see [url=https://github.com/ez-plugins/EzEconomy/blob/main/docs/redis.md]docs/redis.md[/url]) and restart.
	[/list]
[/list]
